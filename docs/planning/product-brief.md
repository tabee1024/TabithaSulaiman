# Product Brief: Tabitha Sulaiman Portfolio

## Project Name

TabithaSulaiman

## Introduction

Hello,

My name is Tabitha Sulaiman. I am a 2026 Computer Science graduate from California State University, Northridge. This portfolio is a full-stack project designed to introduce who I am, how I think, what I build, and how I connect product, design, and engineering decisions.

## Portfolio Purpose

I am building this portfolio as a full-stack MERN application to present myself as a hybrid product, design, and engineering candidate.

The purpose of this portfolio is to showcase my ability to understand user needs, design clear interfaces, build functional software, and explain my decisions using systems thinking.

This portfolio is not just a personal website. I am treating it like production-level software with a defined audience, user journey, feature priorities, technical requirements, security considerations, and future iterations.

## Target Roles

I am tailoring this portfolio toward roles such as:

- Entry-level Product Manager
- UI/UX Developer
- Software Engineer
- Product-minded full-stack developer
- Roles that value product management, UI/UX design, software engineering, or a combination of these skills

## Target Audience

Whether you are a recruiter, hiring manager, technical interviewer, collaborator, mentor, or someone exploring my work, this portfolio is designed to help you quickly understand:

- Who I am
- What types of roles I am targeting
- How I think through problems
- How I design user-centered experiences
- How I build and document software
- How I connect product, design, and engineering decisions
- How I approach projects with systems thinking

## Core Message

I combine product thinking, UX design, and full-stack development to build digital experiences that are clear, useful, and thoughtfully structured.

## Portfolio Personality

I want the visual style to feel personal, warm, structured, creative, and memorable.

The design will be inspired by my Canva mockups, including a cream background, brown interface elements, pixel-style headings, project cards, and playful brand imagery.

The goal is to make the portfolio feel like me while still being polished, readable, and professional for recruiters, hiring managers, technical interviewers, and other visitors.

## MVP Features

For the first working version, I want to build:

- Homepage
- About section
- Projects section
- Project cards
- Basic case study structure
- Contact page
- Responsive layout
- React project data file
- GitHub README
- Basic frontend deployment

## Full-Stack Features

For the full-stack version, I want to add:

- Express backend
- MongoDB database connection
- Contact form API
- Contact messages saved to MongoDB
- Admin dashboard foundation
- Environment variable protection

## Future Features

After the main version works, I may add:

- Admin authentication
- Add/edit/delete projects from dashboard
- Project data stored fully in MongoDB
- Blog or learning notes
- Privacy-friendly analytics
- Accessibility improvements
- More advanced animations

## Security Rules

I will not expose private credentials, API keys, database passwords, MongoDB connection strings, JWT secrets, admin login information, or private tokens in this repository.

Private values will be stored in `.env` files. Safe placeholder examples will be stored in `.env.example` files.

## Privacy Direction

This site should not sell visitor data.

The contact form should only collect information that a visitor voluntarily submits. Contact information should only be used to respond to messages and should not be shared with third parties.
